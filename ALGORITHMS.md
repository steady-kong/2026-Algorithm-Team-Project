# 알고리즘 문서 (Morgorithm)

> 알고리즘 수업 프로젝트 — **"LLM은 생성, 알고리즘이 일한다"**
>
> Upstage Solar LLM은 자연어 요청을 읽고 **후보 데이터(목표 맛 + 커피 후보)** 만 만들어낸다.
> 그 후보 위에서 **점수화·정렬·선택·구성**을 수행하는 것은 전부 직접 구현한 알고리즘이다.
> (표준 `Array.prototype.sort` / 라이브러리 미사용 — 모두 from-scratch.)

전체 흐름:

```
한 줄 요청
  → (LLM 또는 규칙 폴백) 목표 맛 프로파일 + 후보 6개 생성
  → [유사도] 각 후보 점수화
  → [Merge Sort] 적합도 내림차순 정렬
  → [Greedy] 추출방식이 겹치지 않게 다양성 선택 → 최종 3장
  → [이진 탐색 + 0/1 Knapsack DP] 예산 내 시음 세트 구성
  → [Dijkstra + 이진 힙] 기준 커피 → 추천 커피 취향 여정
```

코드 위치: [`website/src/lib/algorithms/`](website/src/lib/algorithms/) · 조합/오케스트레이션은 [`website/src/lib/server/recommend.ts`](website/src/lib/server/recommend.ts).

---

## 1. 가중 L1 유사도 점수 — `similarity.ts`

- **무엇**: 후보의 4축(신맛/단맛/쓴맛/바디감) 프로파일과 사용자의 목표 프로파일 사이 거리를 0~1 적합도로 환산.
- **어떻게**: 축별 절대차(맨해튼/L1 거리)에 가중치를 곱해 합하고, 가능한 최대 거리로 정규화한 뒤 `1 − 정규화거리`. 축이 독립적인 순서형 척도라서 L2(유클리드) 대신 L1을 사용.
- **어디서**: 모든 랭킹 단계의 1차 키. 이후 정렬/그리디/배낭이 이 점수를 그대로 소비한다.
- **복잡도**: 비교당 `O(A)` (A=축 수, 상수 4).

## 2. 정렬 — `sort.ts`

직접 구현한 두 정렬을 제공한다.

- **Merge Sort (안정 정렬)** — **실제 랭킹에 사용**. 적합도 내림차순 정렬 시 동점은 LLM이 제안한 순서를 보존해야 하므로 *안정성*이 필요하다. 보장 `O(n log n)`.
- **Quick Sort (Lomuto 파티션, in-place)** — 비교/자료 목적. 평균 `O(n log n)`, 비안정.
- **어디서**: `recommend.ts`에서 적합도 정렬(merge sort, desc)과 비용 오름차순 정렬(merge sort, asc)에 사용.

## 3. 이진 탐색 — `search.ts`

- **무엇**: 비용 오름차순으로 정렬된 후보에서 "예산 이하" 구간의 경계 인덱스를 `O(log n)`에 찾는다 (`lowerBound`/`upperBound`, C++ `std::lower_bound` 의미).
- **어디서**: `recommend.ts`의 시음 세트 구성 단계 — 배낭 DP를 돌리기 전, 예산을 초과하는 후보를 선형 스캔 없이 잘라낸다.
- **복잡도**: `O(log n)`.

## 4. 그리디 다양성 선택 — `greedy.ts`

- **무엇**: 정렬된 후보에서 최종 3장을 고를 때, 단순 top-3는 비슷한 메뉴(예: 라떼 3개)가 겹친다. 그리디로 *추출방식이 아직 안 나온* 후보 중 최고 점수를 차례로 선택해 다양성을 확보하고, 부족하면 차선으로 채운다.
- **전략**: 매 단계 "제약(중복 방식)을 어기지 않는 지역 최적"을 취하는 고전 그리디 휴리스틱.
- **어디서**: `recommend.ts` 최종 카드 선택(`diversify`), 가성비 원두 필터(`affordableTopK`).
- **복잡도**: 정렬된 입력에 대해 `O(n)`.

## 5. 0/1 배낭 (동적 계획법) — `knapsack.ts`

- **무엇**: "오늘의 시음 세트". 각 커피는 비용(weight)과 적합도(value)를 갖는다. **예산(W)을 넘지 않으면서 적합도 합이 최대가 되는 부분집합**을 고른다 — 각 커피는 최대 1번(0/1).
- **어떻게**: 바텀업 표 `dp[i][w] = 앞 i개로 용량 w에서 얻는 최대 가치`를 채우고 역추적으로 선택 집합 복원. 적합도(실수)는 정수로 스케일링해 정확히 계산.
- **어디서**: `recommend.ts`의 `flight`(시음 세트). about 페이지 `KnapsackDemo`에서 애니메이션으로 시연.
- **복잡도**: `O(n · W)` 시간, `O(n · W)` 공간.

## 6. 다익스트라 최단경로 + 이진 힙 — `graph.ts`

- **무엇**: 맛 공간을 그래프로 모델링한다. 두 커피의 프로파일 L1 거리가 임계값 이하면 간선으로 잇고 가중치 = 그 거리. **기준 커피(중립에 가장 가까운 메뉴)에서 추천 커피까지** 가장 부드럽게(작은 변화의 연속) 이어지는 경로를 다익스트라로 찾는다.
- **자료구조**: 우선순위 큐는 직접 구현한 **이진 최소 힙**(`MinHeap`, bubble up/down).
- **어디서**: `recommend.ts`의 `journey` (취향 여정). about 페이지 `GraphDemo`에서 경로 추적 애니메이션.
- **복잡도**: 그래프 구성 `O(V²)`(메뉴 수 작음), 다익스트라 `O((V+E) log V)`.

## 7. 선형 보간 (블렌드) — `interpolate.ts`

- **무엇**: 두 프로파일을 비율 `t`로 축별 선형 보간 `a·(1−t)+b·t` 후 1~5 격자로 클램프.
- **어디서**: "둘을 섞은 느낌" 액션, 그리고 취향 여정에서 노드 사이를 부드럽게 잇는 중간 프로파일 생성.
- **복잡도**: `O(A)`.

## 자료구조 요약

| 자료구조 | 위치 | 용도 |
|----------|------|------|
| 이진 최소 힙 (Binary Min-Heap) | `graph.ts` `MinHeap` | 다익스트라 우선순위 큐 |
| 인접 리스트 (Adjacency List) | `graph.ts` `TasteGraph` | 맛 공간 그래프 |
| DP 테이블 (2D) | `knapsack.ts` | 0/1 배낭 |

## 테스트

알고리즘은 순수 함수라 단위 테스트로 검증한다.

```bash
cd website && PATH=/usr/local/bin:$PATH npm run test
```

각 알고리즘의 정확성·경계·복잡도 가정은 [`website/src/lib/algorithms/*.spec.ts`](website/src/lib/algorithms/) 참고.
